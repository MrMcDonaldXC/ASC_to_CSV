## 1. 软件概述

### 1.1 项目背景

在现代汽车电子系统开发过程中，CAN（Controller Area Network）总线作为车辆内部通信的核心协议，承载着大量的传感器数据和控制信号。系统集成测试阶段需要采集和分析海量的CAN总线数据，这些数据通常以ASC格式（Vector CANoe/CANalyzer日志格式）存储。

传统的数据处理方式存在以下痛点：

| 痛点 | 描述 |
|------|------|
| **数据格式不兼容** | ASC格式无法直接导入Excel、MATLAB等常用分析工具 |
| **手动转换效率低** | 工程师需要手动编写脚本或使用商业软件进行格式转换 |
| **信号分类困难** | 缺乏智能化的信号分组机制，难以快速定位目标数据 |
| **可视化能力不足** | 缺乏轻量级的数据可视化工具，依赖重型商业软件 |
| **数据对比繁琐** | 多文件数据对比需要复杂的操作流程 |

### 1.2 产品定位

**ASC to CSV 转换与可视化工具** 是一款面向汽车电子系统集成测试工程师的专业数据处理工具，定位于：

```

│                        产品定位矩阵                              │
├─────────────────────────────────────────────────────────────────┤
│  轻量级 ←───────────────────────────────────────────→ 重型软件  │
│     ↑                                                           │

│  通用化 ←───────────────────────────────────────────→ 专业化    │
└─────────────────────────────────────────────────────────────────┘
```

**核心定位特点：**

1. **轻量级部署**：无需安装商业软件，Python环境即可运行
2. **专业化功能**：针对CAN总线数据特点设计，支持DBC解码
3. **智能化分组**：基于命名规则的动态信号分组
4. **一体化流程**：转换、可视化、对比功能集成

#### 3.1.2 DBC加载器 (DBCLoader)

**功能描述：**

DBC加载器负责解析DBC数据库文件，建立CAN消息ID到消息对象的映射关系，为后续的信号解码提供基础数据。

**核心方法：**

| 方法名 | 参数 | 返回值 | 功能描述 |
|--------|------|--------|----------|
| `load(dbc_files)` | DBC文件路径列表 | bool | 加载多个DBC文件 |
| `_load_single_dbc(dbc_file)` | 单个DBC文件路径 | bool | 加载单个DBC文件 |
| `get_message_count()` | 无 | int | 获取消息总数 |
| `get_signal_count()` | 无 | int | 获取信号总数 |

**数据结构：**

```python
message_map = {
    frame_id: {
        'message': Message对象,
        'dbc_name': '文件名.dbc'
    }
}

signal_info = {
    'dbc文件名::消息名::信号名': {
        'unit': '单位',
        'message': '消息名',
        'dbc': 'DBC文件名'
    }
}
```

#### 3.1.3 ASC解析器 (ASCParser)

**功能描述：**

ASC解析器负责解析ASC格式的CAN日志文件，提取CAN帧数据并根据DBC定义解码信号值。

**核心算法：**

1. **编码检测**：自动尝试多种编码（utf-8 → gbk → gb2312 → latin-1）
2. **正则匹配**：使用预编译正则表达式提取CAN帧信息
3. **时间采样**：按配置的采样间隔聚合数据
4. **信号解码**：使用cantools解码CAN帧数据

**正则表达式模式：**

```python
ASC_PATTERN = re.compile(
    r'^(\d+\.\d+)\s+(\d+)\s+([0-9A-Fa-f]+x?)\s+(Rx|Tx)\s+d\s+(\d+)\s+(([0-9A-Fa-f]{2}\s*)+)$'
)
```

**性能优化措施：**

| 优化项 | 实现方式 | 效果 |
|--------|----------|------|
| 缓冲读取 | buffering=8192*4 | 减少I/O次数 |
| 进度更新间隔 | 每10000行更新 | 减少UI开销 |
| 内存检查 | 每50000行检查 | 及时预警 |
| 信号数限制 | MAX_MEMORY_SIGNALS=10000 | 防止内存溢出 |

#### 3.1.4 分组提取器 (GroupExtractor)

**功能描述：**

分组提取器实现基于信号命名规则的动态分组功能，从信号名称中提取组标识符。

**分组规则：**

```
┌─────────────────────────────────────────────────────────────────┐
│                        分组规则定义                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  规则1: BatP + 数字                                             │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 正则: (?i)(BatP\d+)(?=_|$|[^a-zA-Z0-9])                  │   │
│  │ 匹配: BatP1, BatP10, BatP28, batp3, BATP5               │   │
│  │ 输出: BATP1, BATP10, BATP28, BATP3, BATP5               │   │
│  └─────────────────────────────────────────────────────────┘   │
```

**分段绘制算法：**

```python
def plot_segments(x_data, y_data, max_gap=2):
    """
    分段绘制算法：
    1. 找出所有有效数据点索引
    2. 根据max_gap分割成多个连续段
    3. 分别绘制每个段
    """
    valid_indices = [i for i, v in enumerate(y_data) if v is not None]
    segments = []
    current_segment = [valid_indices[0]]
    
    for i in range(1, len(valid_indices)):
        if valid_indices[i] - valid_indices[i-1] <= max_gap:
            current_segment.append(valid_indices[i])
        else:
            segments.append(current_segment)
            current_segment = [valid_indices[i]]
    
    # 分别绘制每个段
    for segment in segments:
        plot(segment_x, segment_y)
```
