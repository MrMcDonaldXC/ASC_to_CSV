## 一、项目概述

### 2.2 核心功能详解

#### 2.2.1 数据转换功能

| 功能项 | 描述 | 技术实现 |
|--------|------|----------|
| **ASC文件解析** | 支持Vector标准ASC格式 | 正则表达式匹配 |
| **多DBC支持** | 同时加载多个DBC数据库文件 | cantools库 |
| **信号解码** | 根据DBC定义解码CAN帧数据 | 自动类型转换 |
| **智能分组** | 按BatP命名规则自动分组 | 正则表达式提取 |
| **空值填充** | 自动填充缺失数据点 | 时间区间填充算法 |
| **采样聚合** | 按时间间隔聚合数据 | 时间窗口采样 |

#### 2.2.2 智能分组规则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        信号分组规则说明                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  规则1: BatP + 数字                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 匹配模式: BatP1, BatP2, BatP3, ..., BatP10, BatP28, ...            │   │
│  │ 输出文件: BATP1.csv, BATP2.csv, BATP3.csv, ...                     │   │
│  │ 示例信号: 800V_BMS.dbc::BatP3_BMS_CellVolt::P3_AvgCellVlt          │   │
│  │          → 分组: BATP3 → 输出: BATP3.csv                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  规则2: BatP + 字母                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 匹配模式: BatPS, BatPQ, BatPL, BatPR, BatPE, ...                   │   │
│  │ 输出文件: BATPS.csv, BATPQ.csv, BATPL.csv, ...                     │   │
│  │ 示例信号: test.dbc::BatPS_Status::Signal                           │   │
│  │          → 分组: BATPS → 输出: BATPS.csv                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  规则3: 其他信号                                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 匹配模式: 不符合以上规则的信号                                      │   │
│  │ 输出文件: Others.csv                                                │   │
│  │ 示例信号: test.dbc::FMC_Msg::Signal                                │   │
│  │          test.dbc::HVMS_Msg::Signal                                │   │
│  │          → 分组: Others → 输出: Others.csv                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  注意事项:                                                                  │
│  • 分组名称大小写不敏感（BatP3 和 BATP3 归入同一组）                        │
│  • 数字支持多位数（BatP1 和 BatP10 是不同分组）                             │
│  • 字母支持1-2位（BatPS, BatPQ, BatPL 等）                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 输入输出文件

#### 2.3.1 输入文件格式

| 文件类型 | 扩展名 | 格式说明 |
|----------|--------|----------|
| **ASC文件** | .asc | Vector CANoe/CANalyzer日志格式 |
| **DBC文件** | .dbc | CAN数据库定义文件 |

**ASC文件格式示例：**

```
date Sat Jan 01 10:00:00 2024
base hex timestamps absolute
no internal Events logged
Begin Triggerblock Sat Jan 01 10:00:00 2024
   0.000000 1  123             Rx   d 8 01 02 03 04 05 06 07 08
   0.100000 1  456             Rx   d 8 AA BB CC DD EE FF 00 11
   0.200000 2  789x            Rx   d 8 12 34 56 78 9A BC DE F0
End TriggerBlock
```

#### 2.3.2 输出文件格式

| 文件名 | 内容描述 |
|--------|----------|
| **Summary.csv** | 转换汇总报告，包含统计信息 |
| **BATP1.csv** | BATP1分组的所有信号数据 |
| **BATP2.csv** | BATP2分组的所有信号数据 |
| **...** | 其他分组文件 |
| **Others.csv** | 未匹配分组规则的信号数据 |

**CSV输出格式示例：**

```csv
Time[s],BMS.dbc::BatP3_Status::PackSOC[%],BMS.dbc::BatP3_Status::PackVolt[V]
0.0,80.5,400.2
0.1,80.4,400.1
0.2,80.3,400.0
```

#### 4.4.2 配置文件格式

```json
{
    "asc_file": "data/input/test_20260325.asc",
    "dbc_files": [
        "dbc/BMS.dbc",
        "dbc/VCU.dbc",
        "dbc/MCU.dbc"
    ],
    "output_dir": "data/output",
    "sample_interval": 0.1,
    "group_size": 5,
    "csv_encoding": "utf-8-sig",
    "debug": false
}
```

#### 4.4.3 配置参数详解

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `asc_file` | 字符串 | "" | ASC日志文件的完整路径 |
| `dbc_files` | 数组 | [] | DBC文件路径列表，支持多个文件 |
| `output_dir` | 字符串 | "" | CSV文件的输出目录 |
| `sample_interval` | 数值 | 0.1 | 采样间隔（秒），建议范围0.01-1.0 |
| `group_size` | 整数 | 5 | 分组大小，用于空行插入 |
| `csv_encoding` | 字符串 | "utf-8-sig" | CSV文件编码，推荐utf-8-sig |
| `debug` | 布尔 | false | 是否启用调试模式 |

#### 4.4.4 编码选择建议

| 编码 | 适用场景 | Excel兼容性 |
|------|----------|-------------|
| **utf-8-sig** | 推荐，带BOM的UTF-8 | ✓ 完美兼容 |
| **utf-8** | 标准UTF-8编码 | △ 可能乱码 |
| **gbk** | 中文Windows系统 | ✓ 兼容 |
| **gb2312** | 老旧系统 | △ 部分兼容 |

## 六、常见问题解答（FAQ）

### 6.1 安装与启动问题

**Q1: 双击EXE文件无反应怎么办？**

```
A: 可能原因及解决方法:

1. 杀毒软件拦截
   解决: 将程序添加到杀毒软件白名单
   
2. 缺少运行库
   解决: 安装 Microsoft Visual C++ Redistributable
   下载地址: https://aka.ms/vs/17/release/vc_redist.x64.exe
   
3. 文件损坏
   解决: 重新下载程序文件
```

**Q2: 源码版本运行报错"ModuleNotFoundError"？**

```
A: 依赖包未安装，请执行:

pip install -r requirements.txt

如果下载速度慢，可使用国内镜像:

pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**Q3: 程序启动后界面显示乱码？**

```
A: 系统字体问题，解决方法:

1. 安装中文字体
   • SimHei（黑体）
   • Microsoft YaHei（微软雅黑）

2. 或修改源码中的字体设置
   文件: main_app.py
   修改: plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei']
```

### 6.2 数据转换问题

**Q4: 为什么有些信号没有解析出来？**

```
A: 可能原因:

1. DBC文件中未定义该消息
   解决: 确认DBC文件包含所需消息定义
   
2. 消息ID不匹配
   解决: 检查ASC文件中的消息ID是否在DBC中定义
   
3. 信号名称不匹配
   解决: 启用调试模式查看详细错误信息

排查方法:
1. 勾选"调试模式"
2. 执行转换
3. 查看日志中的错误信息
```

**Q5: 转换时提示"内存不足"怎么办？**

```
A: 解决方法:

1. 增大采样间隔
   • 当前: 0.1秒
   • 建议: 0.5秒或1.0秒
   
2. 分段处理大文件
   • 将大ASC文件分割成多个小文件
   • 分别转换后合并CSV
   
3. 关闭其他程序
   • 释放系统内存
   
4. 升级硬件
   • 建议内存: 8GB以上
```

**Q6: 转换后的CSV文件为什么有很多空值？**

```
A: 这是正常现象，原因:

1. 信号采样频率不同
   • 不同信号的发送周期不同
   • 某些时刻可能没有数据更新
   
2. 解决方法:
   • 软件已自动进行空值填充
   • 填充间隔默认0.5秒
   • 可在代码中调整 fill_interval 参数
```

**Q7: 分组结果不符合预期怎么办？**

```
A: 检查分组规则:

1. 当前分组规则:
   • BatP + 数字 → BATP1, BATP2, BATP10...
   • BatP + 1-2个字母 → BATPS, BATPQ, BATPL...
   • 其他 → Others
   
2. 常见问题:
   • 大小写: BatP3 和 BATP3 归入同一组（正确）
   • 多位数: BatP1 和 BatP10 是不同分组（正确）
   
3. 自定义分组:
   • 修改 group_extractor.py 中的正则表达式
   • 正则: r'(?i)(BatP\d+|BatP[A-Z]{1,2})(?=_|$|[^a-zA-Z0-9])'
```

### 6.3 数据可视化问题

**Q8: 图表无法显示？**

```
A: 检查以下项:

1. CSV文件是否正确加载
   • 查看文件下拉框是否有文件列表
   • 尝试刷新文件列表
   
2. 数据列是否有效
   • 确认选择的列是数值类型
   • 非数值列无法绑制图表
   
3. 数据是否为空
   • 打开CSV文件确认有数据
   • 检查是否有有效的数值
```

**Q9: 图表显示卡顿怎么办？**

```
A: 优化方法:

1. 缩小查看范围
   • 使用缩放功能查看局部数据
   
2. 增大采样间隔
   • 重新转换时使用更大的采样间隔
   
3. 关闭其他程序
   • 释放系统资源
```

**Q10: 十字参考线坐标不准确？**

```
A: 可能原因:

1. 图表缩放后坐标系变化
   解决: 重置视图后重新启用十字参考线
   
2. 鼠标移动过快
   解决: 慢速移动鼠标
```

### 6.4 数据对比问题

**Q11: 对比图显示异常？**

```
A: 检查以下项:

1. 文件格式是否一致
   • 确保对比的文件有相同的列结构
   
2. 数据范围是否差异过大
   • 不同数量级的数据可能显示不明显
   
3. 时间轴是否对齐
   • 不同长度的数据可能显示不完整
```

**Q12: 如何导出对比图？**

```
A: 当前版本暂不支持直接导出，替代方法:

1. 使用截图工具
   • Windows: Win + Shift + S
   • 或使用第三方截图软件
   
2. 未来版本将支持:
   • 导出PNG/JPG图片
   • 导出PDF文件
```

### 7.3 常见故障排除

#### 7.3.1 转换失败

```
症状: 点击"开始转换"后提示错误或无响应

排查步骤:
1. 检查文件路径
   □ ASC文件是否存在
   □ DBC文件是否存在
   □ 输出目录是否有效

2. 检查文件权限
   □ 是否有读取权限
   □ 是否有写入权限

3. 检查文件内容
   □ ASC文件格式是否正确
   □ DBC文件是否有效
   □ 文件是否损坏

4. 检查系统资源
   □ 内存是否充足
   □ 磁盘空间是否充足
```

#### 7.3.2 数据不完整

```
症状: 转换后的CSV文件数据缺失

排查步骤:
1. 检查DBC覆盖范围
   □ DBC是否包含所有消息定义
   □ 是否需要添加更多DBC文件

2. 检查采样间隔
   □ 采样间隔是否合适
   □ 是否需要调整

3. 检查数据源
   □ ASC文件是否完整
   □ 数据时间范围是否正确
```

#### 7.3.3 性能问题

```
症状: 程序运行缓慢或卡顿

排查步骤:
1. 检查数据量
   □ 文件是否过大（>500MB）
   □ 信号数量是否过多（>1000）

2. 优化建议
   • 增大采样间隔: 0.1 → 0.5 → 1.0
   • 分段处理大文件
   • 关闭不必要的程序

3. 硬件升级
   • 内存升级到8GB以上
   • 使用SSD硬盘
```

### 9.3 文件格式参考

#### 9.3.1 ASC文件格式详解

```
ASC文件结构:

文件头:
────────────────────────────────────────
date Sat Jan 01 10:00:00 2024        ← 日期
base hex timestamps absolute         ← 数制和时间戳类型
no internal Events logged            ← 内部事件设置
Begin Triggerblock Sat Jan 01 10:00:00 2024
────────────────────────────────────────

数据行格式:
────────────────────────────────────────
   0.000000 1  123             Rx   d 8 01 02 03 04 05 06 07 08
   │        │  │               │    │ │ └───────────────────── 数据字节
   │        │  │               │    │ └── 数据长度
   │        │  │               │    └── 方向标识(d=数据帧)
   │        │  │               └── 收发方向(Rx=接收, Tx=发送)
   │        │  └── 帧ID(十六进制，x表示扩展帧)
   │        └── 通道号
   └── 时间戳(秒)
────────────────────────────────────────

文件尾:
────────────────────────────────────────
End TriggerBlock
────────────────────────────────────────
```

#### 9.3.2 CSV输出格式详解

```
CSV文件结构:

表头行:
────────────────────────────────────────
Time[s],BMS.dbc::BatP3_Status::PackSOC[%],BMS.dbc::BatP3_Status::PackVolt[V]
│      │                                  │
│      │                                  └── 信号名称[单位]
│      └── DBC文件名::消息名::信号名[单位]
└── 时间列
────────────────────────────────────────

数据行:
────────────────────────────────────────
0.0,80.5,400.2
0.1,80.4,400.1
0.2,80.3,400.0
│   │     │
│   │     └── 信号值
│   └── 信号值
└── 时间戳(秒)
────────────────────────────────────────
```
